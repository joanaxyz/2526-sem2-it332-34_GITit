The user (designer's eye) wants the GIT-IT practice UI kept minimal and on-theme, not information-dense.

**Why:** Earlier challenge/adventure screens were "too wordy" and "information overloaded."

**How to apply:**
- Top navbars: cut redundant labels. Removed the "Storey N / <title>" line and the "Working change" variant badge from the challenge header ([ChallengeStatusHeader.tsx](frontend/src/features/challenges/components/ChallengeStatusHeader.tsx)); keep the command-budget "scaffolding" and action buttons only. Left a compact difficulty badge.
- Live/Target DAG ([LiveDagPanel.tsx](frontend/src/shared/practice/components/LiveDagPanel.tsx)): show commit nodes + ref pills only. Dropped the `showRepositoryDetails` metadata grid in the challenge workspace and trimmed the hover overlay to hash + message + refs + HEAD/merge (no changed-paths/tree dumps).
