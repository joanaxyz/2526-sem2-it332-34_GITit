# Plan — Tower-Defense Battle Redesign

Branch: `chore/codebase-improvements`. Replaces the horizontal parallax battle with a
**vertical tower climb** that reuses the `/tower` page's separators, sky, and Blue
character. "Protect the tower": monsters besiege a crystal on each level; the player
clears them by solving the git task, then flies up to the next level.

## Locked decisions (from the user)
1. **Full backend rename, no legacy.** Adventure's per-task unit `AdventureQuest` →
   `AdventureLevel` (levels live under `CommandAdventure`, which IS "the adventure's quest").
   `ChallengeQuest` stays (the difficulty = the quest); each challenge quest wraps exactly
   **one** boss level (the single separator the player stands on).
2. **Monster/boss HP = number of evaluation rules to reach `target_state`.** Each newly
   satisfied rule = 1 hit; solving lands the finishing blow. No `min_counted_commands`-derived
   HP, no `BOSS_HP_PAR_BONUS`/`MAX_DEFAULT_MONSTERS` magic numbers.
3. **One resource: mana.** Mana pool = `max_counted_commands`. Every counted command costs
   1 mana. **Misses are purely visual** (monster lunges at the crystal; player plays a "miss"
   action — dupe `idle` until a real sprite exists). **No crystal HP.** Crystal breaks when
   **mana hits 0** before the level is solved → `player_defeat`.

## Structure (target)
- **Adventure** = `CommandAdventure` (the single "quest") → many `AdventureLevel`s, climbed in
  one session via the existing mastery scheduler (order + rotation preserved). Each level =
  one separator with a monster pack + crystal. Progress/stats/diagnostics persist even if the
  session is abandoned mid-climb (already true on `AdventureRun`/`AdventureMastery`; keep).
- **Challenge** = `Challenge` → many `ChallengeQuest`s (easy/medium/hard), each a **separate
  session** = one separator with a **boss** + crystal. Solve the scenario → kill the boss.

### B. HP from evaluation rules (replaces par-based + boss-bonus magic)
- Add `backend/evaluation/…` helper `count_target_rules(evaluation_spec) -> int` (the total
  number of state/process rules the engine evaluates toward `target_state`). Reuse whatever
  `EvaluationEngine` already enumerates so HP and `len(passed_rules)` share one definition.
- `backend/battle/state.py`:
  - `initial_adventure_battle_state`: total roster HP = `count_target_rules(level.evaluation_spec)`
    (fallback 1 if 0). Distribute across `min(rule_count, MAX_DEFAULT_MONSTERS)` mobs (keep the
    multi-monster pack; drop par as the HP source). `encounter_spec` still overrides.
  - `initial_challenge_battle_state`: boss HP = `count_target_rules(quest.evaluation_spec)`
    (fallback ≥1). Drop `BOSS_HP_PAR_BONUS`. `boss_spec` still overrides.
- `backend/battle/constants.py`: delete `BOSS_HP_PAR_BONUS`; keep `MAX_DEFAULT_MONSTERS` only
  as a pack-size cap (no longer HP-linked). Update the docstring.
- `backend/battle/engine.py` `_hit`: deal `delta = rules_passing - previous_rules` damage
  (≥1) instead of a flat 1, so total hit damage == rule count == HP and the monster dies
  exactly as the repo reaches target (finishing blow then no-ops but still emits
  `encounter_cleared`). Update `battle/tests/test_engine.py`/`test_turn.py`.

### C. Mana (rename the "budget" framing; behavior already matches)
- Mana pool = `max_counted_commands`; cost = 1 per counted command; `defeated = (not solved
  and counted and counted_total >= max_counted_commands)` — already computed in
  `adventures/services.py` + `practice/services.py`. Keep the mechanic; rename payload framing
  from "budget" to "mana" where it surfaces (`counts`/`policy` blocks) **only** if we want the
  wire to say mana — otherwise keep keys and relabel in the FE. Recommend: keep wire keys,
  relabel in FE to minimize churn (decide during impl; note in PR).

### D. Seed / authoring
- `seed_curriculum_v2.py` + `curriculum_v2/adventure_quests.py`/`challenges.py`: no HP authoring
  needed anymore (HP is derived). Keep `encounter_spec`/`boss_spec` as optional overrides.
  Remove any now-dead HP/par authoring keys if present. Reseed local sqlite after migration.
